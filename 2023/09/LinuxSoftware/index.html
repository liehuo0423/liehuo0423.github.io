Linux基础软件安装

JDK1.8安装

/usr/local/java/jdk1.8.0_381

1、使用Xftp把 jdk-8u381-linux-x64.tar.gz 文件传入 /usr/local/java 目录下

$ mkdir /usr/local/java

2、解压

$ tar -zxvf jdk-8u381-linux-x64.tar.gz

3、配置环境变量

$ vim /etc/profile
...
export JAVA_HOME=/usr/local/java/jdk1.8.0_381
export PATH=$JAVA_HOME/bin:$PATH

4、让配置文件生效

$ source /etc/profile
$ java -version

Tomcat安装

/opt/software/tomcat/apache-tomcat-9.0.78

1、上传安装文件，并解压缩到/opt/tomcat
2、进入解压目录 /bin ，启动tomcat ./startup.sh
3、开放端口 8080

#开启防火墙
$ systemctl start firewalld
#添加端口
$ firewall-cmd --add-port=8080/tcp --permanent
success
#重新加载防火墙规则
$ firewall-cmd --reload
success
$ firewall-cmd --query-port=8080/tcp
yes
#查看所有永久开放的端口
$ firewall-cmd --list-ports --permanent
#查看防火墙状态
$ systemctl status firewalld

/usr/local/mysql

CentOS 安装MySql8 离线手动安装 （rpm） - 乐帮网

8、修改密码

#首次修改密码
$ mysql> ALTER user 'root'@'localhost' IDENTIFIED BY '123456';

#修改远程密码
$ mysql> ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY 'xxx';

9、开启远程访问

$ mysql -u root -p ${your password}

$ use mysql;
$ select host, user, authentication_string, plugin from user;

#如果没有host=%的，说明没有启用远程登陆的账户
$ update user set host='%' where user='root';
#给远程账户授权
$ Grant all privileges on root.* to 'root'@'%';
#设置密码
$ ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '${your password}';
#刷新权限
$ flush privileges;

10、开启防火墙 3306 端口的访问权限

$ firewall-cmd --zone=public --add-port=3306/tcp --permanent
#重启防火墙
$ firewall-cmd --reload
#查看mysql版本
$ mysql -V

Nginx安装

1、编译环境

$ yum install git gcc gcc-c++ make automake autoconf libtool pcre pcre-devel zlib zlib-devel openssl-devel wget vim -y

or
安装所需插件

  1. gcc

gcc是linux下的编译器，它可以编译 C,C++,Ada,Object C和Java等语言

#查看gcc版本 
$ gcc -v

$ yum -y install gcc
  1. pcre、pcre-devel

pcre是一个perl库，包括perl兼容的正则表达式库，nginx的http模块使用pcre来解析正则表达式

$ yum install -y pcre pcre-devel
  1. zlib

zlib库提供了很多种压缩和解压缩方式，nginx使用zlib对http包的内容进行gzip

$ yum install -y zlib zlib-devel
  1. openssl

openssl是web安全通信的基石

$ yum install -y openssl openssl-devel

2、把nginx压缩包解压到 /usr/local/nginx 下

$ tar -zxvf nginx-1.20.2.tar.gz

3、执行./configure并进行编译安装

$ cd /usr/local/nginx/nginx-1.20.2/
#执行
$ ./configure
#编译安装
$ make && make install

4、启动nginx服务

$ cd /usr/local/nginx/sbin
$ ./nginx 
# or
$ /usr/local/nginx/sbin/nginx

#启动nginx
$ /usr/local/nginx/sbin/nginx
 #重启nginx
$ /usr/local/nginx/sbin/nginx -s reload
#停止nginx
$ /usr/local/nginx/sbin/nginx -s stop

检查配置文件是否有误

#编辑配置文件
$ vim /usr/local/nginx/conf/nginx.conf
#检测配置文件
$ ./nginx -t

ngx_http_fastdfs_set pid=29157
nginx: the configuration file /usr/local/nginx/conf/nginx.conf syntax is ok
nginx: configuration file /usr/local/nginx/conf/nginx.conf test is successful

5、查看nginx进程

$ ps aux | grep nginx
#or
netstat -ntpl

6、开启端口

[root@liehuo sbin]$ firewall-cmd --add-port=80/tcp --permanent
success
[root@liehuo sbin]$ firewall-cmd --reload
success

Redis安装

/usr/local/redis

1、把redis压缩包解压到 /usr/local/redis 下

$ tar -zvxf redis-5.0.5.tar.gz

2、使用make命令进行编译

$ cd /usr/local/redis/redis-5.0.5
$ make

3、进入src文件执行make install安装redis

$ cd /usr/local/redis/redis-5.0.5/src
$ make install

4、启动redis（前台）

[root@liehuo src]$ ./redis-server ../redis.conf

5、修改redis.conf配置文件

#redis server 接收这个网卡过来的连接
bind 192.168.221.129
# 允许任何主机连接、访问
bind 127.0.0.1 改为 bind 0.0.0.0

// 修改保护模式, 不修改保护模式也是只能内网访问的 protected-mode yes 改成 protected-mode no
protected-mode no
// daemonize no 改为yes 后台一直运行
daemonize yes
// 设置密码, 这里建议设置密码, 否则可能会发生一些预料不到的事情, 因为6379端口有漏洞
requirepass "你的密码"

6、后台运行

$ ./redis-server ../redis.conf

7、查看redis进程

$ ps -ef | grep redis

8、开放端口

firewall-cmd --zone=public --add-port=6379/tcp --permanent
firewall-cmd --reload

9、启动redis-cli

[root@liehuo src]$ ./redis-cli
127.0.0.1:6379> auth 123456
OK
127.0.0.1:6379> keys *
(empty list or set)
127.0.0.1:6379>

RocketMQ安装

/usr/local/rocketmq/rocketmq-4.9.2/

rocketmq 需要 Linux 上安装JDK，版本 1.8 及以上
1、上传之后进行解压:

unzip rocketmq-all-4.7.0-bin-release.zip

目录结构

  • benchmark：测试 sh 文件
  • bin：可执行文件
  • conf：配置文件
  • lib：mq 所依赖的第三方 jar 包

2、启动 RocketMQ

在 bin 目录下，执行下面命令
启动nameserver

cd /usr/local/rocketmq/rocketmq-4.9.2/bin
#后台运行
[root@liehuo bin]$ nohup sh mqnamesrv &

[1] 4456
nohup: ignoring input and appending output to 'nohup.out'

启动 broker

#后台运行
[root@liehuo bin]$ nohup sh mqbroker -c /usr/local/rocketmq/rocketmq-4.9.2/conf/broker.conf &

[1] 7940
nohup: ignoring input and appending output to 'nohup.out'

#or 前台运行
[root@liehuo rocketmq-4.9.2]$ bin/mqnamesrv
[root@liehuo rocketmq-4.9.2]$ bin/mqbroker -c /usr/local/rocketmq/rocketmq-4.9.2/conf/broker.conf

# 查看日志，看 mq 是否启动成功
[root@liehuo bin]$ tail -f ~/logs/rocketmqlogs/broker.log
[root@liehuo rocketmq]$ tail -f ~/logs/rocketmqlogs/namesrv.log

3、进入 conf 目录追加内容

$ cd /usr/local/rocketmq/rocketmq-4.9.2/conf
$ vim broker.conf

#追加下面内容
listenPort=10911
namesrvAddr=localhost:9876
storePathRootDir=/usr/local/rocketmq/rocketmq-4.9.2/store
storePathCommitLog=/usr/local/rocketmq/rocketmq-4.9.2/store/commitlog
storePathConsumerQueue=/usr/local/rocketmq/rocketmq-4.9.2/store/consumequeue

4、修改内存大小
不修改内存大小可能运行不起来
进入 bin 目录
编辑 runbroker.sh 文件和 runserver.sh 文件，修改内存大小

vim runserver.sh

vim runbroker.sh

5、查看进程

netstat -an | grep 9876

6、开放防火墙 9876 和 10911 端口

firewall-cmd --zone=public --add-port=9876/tcp --permanent
firewall-cmd --zone=public --add-port=10911/tcp --permanent
firewall-cmd --reload

7、关闭MQ

cd /usr/local/rocketmq/rocketmq-4.9.2/bin
# 关闭 namesrv
sh mqshutdown namesrv

# 关闭 broker
sh mqshutdown broker

Docker安装

#工作目录
/var/lib/docker

#查看已安装的docker文件
yum list installed | grep docker

#更新yum
yum update
# yum-util 提供yum-config-manager功能，另外两个是devicemapper驱动依赖的	
yum install -y yum-utils device-mapper-persistent-data lvm2
#设置yum源
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
#or
yum-config-manager \
    --add-repo \
    http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

#更新yum软件包索引
yum makecache fast

#查看可安装的副本
yum list docker-ce --showduplicates | sort -r
yum list docker-ce-cli --showduplicates | sort -r
yum list containerd.io --showduplicates | sort -r

#安装dokcer docker-ce社区 ee 企业版
yum install docker-ce docker-ce-cli containerd.io
#or
yum install docker-ce-24.0.5-1.el7 docker-ce-cli-24.0.5-1.el7 containerd.io-6.21-3.1.el7

#启动docker
systemctl start docker
#使用docker version 是否安装成功！
#拓展
[root@liehuo ~]# systemctl reload docker
[root@liehuo ~]# systemctl status docker

#卸载依赖
yum remove docker-ce docker-ce-cli containerd.io
#or
yum remove docker-ce-24.0.5-1.el7 docker-ce-cli-24.0.5-1.el7 containerd.io-6.21-3.1.el7

#删除资源
rm -rf /var/lib/docker
#/var/lib/docker docker的默认工作路径！

docker ps -a #运行记录
docker ps #运行时
docker start 99c9b77227de
docker stop 99c9b77227de

#启动并加入开机启动
systemctl start docker
systemctl enable docker

进入etc/docker目录下 [非必要操作]

# ls查看目录下有无daemon.json文件，如果没有创建一个
$ cd /etc/docker
$ ls
daemon.json

进入文件后，添加:

{
  "registry-mirrors": ["https://registry.docker-cn.com"]
}

还可以加入阿里云的docker 镜像加速地址：如 https://y4xpdpoy.mirror.aliyuncs.com
阿里云容器镜像服务 ：https://cr.console.aliyun.com/cn-hangzhou/instances/mirrors

{
  "registry-mirrors": ["https://registry.docker-cn.com","https://y4xpdpoy.mirror.aliyuncs.com"]
}

重启docker服务

service docker restart

elasticsearch docker 无法启动问题

#-e "discovery.type=single-node"这个东西好像是说设置单机使用
#-e ES_JAVA_OPTS="-Xms256m -Xmx256m"
#es启动需要2g的虚拟内存，正常咱们装的虚拟机根本没有分配到这么大的虚拟内存所以要这么写
docker run -d --name es -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" -e ES_JAVA_OPTS="-Xms256m -Xmx256m" 9cd5b752c86e

# 最大虚拟内存区太小了,设置大点
sysctl -w vm.max_map_count=262144
# 然后再启动

将docker镜像拷贝到其他电脑运行

执行如下命令找到被打包镜像的名字和版本号 (版本号=TAG)
docker images
docker打包镜像的两种方式 (选一种执行即可)

docker save 镜像名字:版本号 > /root/打包名字.tar
docker save -o /root/打包名字.tar 镜像名字:版本号

docker save docker.elastic.co/kibana/kibana:7.15.2 > /root/kibana_7.15.2.tar

将打包镜像分发到其它宿主机的 /root/ 目录下
xftp文件传输

#将打成tar包的镜像load出来
docker load < /root/kibana_7.15.2.tar

查看 load 出来的镜像ID
docker images
刚 load 出来的镜像其名字、版本号均为 none, 我们要通过 tag 命令赋予名字和版本号
docker tag 镜像ID 镜像名字:版本号
